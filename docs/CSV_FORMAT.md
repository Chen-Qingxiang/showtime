# CSV 格式与导入质检

## 主时间线

ShowTime 的主时间线 CSV 始终只有两列：

```csv
time,title
```

图层名、角色、颜色、分组、来源和详细说明不进入主 CSV；它们由文件名、项目 Manifest 和可选 sidecar 管理。这使最小 CSV 长期保持可读、可移植，并兼容旧数据。

表头可省略。空行与以 `#` 开头的注释行会被忽略并计入质检统计。

## 时间语法

### 年份

- 公元年：`1949`、`2026 CE`、`公元2026`
- 负数年份：`-221`
- BCE：`221 BC`、`221 BCE`、`公元前221`
- 极大年份：`-13800000000`

`221 BCE` 按无公元 0 年的天文学计年映射到内部值 `-220`；直接写 `-221` 则保留为 `-221`。导出保留用户原始 `time` 文本，不会把一种写法强制改成另一种。

### 日期和日内时间

```text
1949-10
1949-10-01
1969-07-20 20:17
1969-07-20T20:17:40
```

日期目前只支持公元正年份。日期会检查月份、闰年、日、小时、分钟和秒。

### 区间

推荐使用 `~`：

```text
960~1127
1914-07-28~1918-11-11
1969-07-20 20:17~1969-07-20 20:18
1949~
```

兼容分隔符：`~ – — － - 〜 ～ 至 到`。为了避免日期连字符与区间混淆，生成数据时应使用 `~`。

开区间会按起点精度补到当前年、月、日或时间，并产生 `open_interval` 提示。反向区间会规范化为先早后晚，并产生 `reversed_interval` 警告。

## 标题和 CSV 引号

标题含逗号时用英文双引号包住；标题内的双引号写成两个双引号：

```csv
1942,"《春望》, 杜甫相关研究"
1967,"论文 ""The Medium is the Message"" 出版"
```

UTF-8、UTF-8 BOM、LF、CRLF 和 CR 均支持。不支持跨多行的 quoted field。

## 点事件显示

主数据中的点事件不会被改写。显示菜单可以只在视觉上把点扩展为对应精度的时间段：

- 保持圆点
- 年点显示为全年
- 年 / 月 / 日点显示到对应完整精度
- 全部点显示为对应时间单位

搜索、探针和导出仍使用事件的真实时间，不会把视觉扩展误当成原始区间。

## 结构化质检

每次导入都会生成包含文件数、总行数、成功事件、注释、空行和问题列表的报告。有效行会继续导入；问题不会静默消失。

| 类型 | 含义 |
| --- | --- |
| `quote` | 引号没有闭合 |
| `columns` | 缺少标题列 |
| `comma` | 多于两列，通常是标题逗号未加引号 |
| `invalid_date` | 时间无法解析或日期无效 |
| `reversed_interval` | 终点早于起点，已规范化 |
| `open_interval` | 终点随当前时间变化 |
| `empty_title` | 标题为空，已使用“未命名事件” |
| `long_title` | 标题超过 120 字符 |
| `duplicate_event` | 同一图层中 time + title 重复 |
| `mixed_precision` | 同一文件混合年、月、日或日内精度 |
| `duplicate_layer` | 项目中存在重名图层 |
| `sidecar_unmatched` | sidecar 没有匹配事件 |
| `sidecar_ambiguous` | sidecar 匹配到多个事件 |
| `manifest_missing_file` | Manifest 引用文件不存在 |
| `unsupported_version` | 项目版本高于当前支持版本 |

报告可在右栏查看原始行、修复建议，并导出为 Markdown。
